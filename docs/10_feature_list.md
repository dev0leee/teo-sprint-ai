# Step 10. 기능 나열

> 목적: 지금까지 논의에서 나온 **기능 아이디어를 하나도 빠뜨리지 않고** 모은다.

> **Guide:**  
> 1. Agent가 `docs/04`~`docs/09`을 읽고, 언급된 **모든 기능**을 추출한다.  
> 2. **순서·중요도·선후 관계 없이** 일렬로 나열한다.  
> 3. 팀이 빠진 기능이 있으면 **팀이 추가한 기능**에 적는다.  
> - 기능은 **사용자가 할 수 있는 일** 또는 **서비스가 제공하는 것** 단위로 쓴다. (예: 로그인, 일정 자동 추천, 알림 보내기)  
> - 큰 덩어리(예: "앱 전체")로만 적혀 있어도 **일단 포함**하고, 안에 여러 기능이 섞여 있으면 **풀어서** 각각 적는다.  
> - 비슷한 기능이라도 **논의에서 따로 나왔으면 각각** 적는다.  
> - **이 단계에서 하지 않는 것:** 우선순위 정하기, 중요도 판단, MVP만 골라내기, 제외·합치기  
> - 우선순위·제외는 **Step 14**에서 한다. 여기서는 **놓치지 않고 모으는 것**만 한다.

**프롬프트 예시**

```
docs/04~09을 읽고 docs/10_feature_list.md의 "Agent가 추출한 기능" 섹션만 채워줘.

규칙:
- docs/04~09 어디에든 나온 기능·아이디어를 빠짐없이 포함 (목적·가치, 여정, Pain/Wow, 가설, 키워드, 질문·답변 등)
- F-01부터 번호를 매겨 일렬로 나열 (번호·순서는 우선순위가 아님)
- 기능명은 짧고 구체적으로 (한 줄)
- 우선순위 판단·제외·MVP 선별 하지 않는다. 의심스러워도 일단 나열
- "팀이 추가한 기능" 섹션은 건드리지 않는다.
- 구조와 상단 Guide는 유지
```

## Agent가 추출한 기능

> `docs/04`~`docs/09`에서 나온 기능. **빠짐없이** 나열. Agent가 채운다.

**온보딩·개념**

- **F-01** 추구미/비전보드/무드보드 개념을 직관적으로 알려주는 온보딩
- **F-02** 시작 시 간단한 취향 테스트로 첫 보드 자동 세팅
- **F-03** 완성된 무드보드 예시 중 하나를 골라 시작(엔드 이미지 상상 유도)

**질문·취향 입력**

- **F-04** 질문·선택 기반으로 취향을 파악해 무드보드 생성(키워드 검색 없이)
- **F-05** 비전 여정/추구생 여정/추구미 여정 등 '여정(test)' 선택
- **F-06** "나" 모드 질문(내면적·솔직한 톤, AI 심리상담사식 심층 질문)
- **F-07** "타인(친구)" 모드 질문(추측·관찰 재미, 무례하지 않게)
- **F-08** 질문마다 취향 태그(#감성적인 #차분한 등) 부여
- **F-09** 큰 갈래 → 점점 구체화되는 연쇄 질문 흐름
- **F-10** 키워드 검색 방식으로 취향 입력(글자/UI·그림 선택)
- **F-11** 질문 답변 → 사진 자동 매칭

**이미지 확보**

- **F-12** 감성 이미지 자체 큐레이션 풀 제공
- **F-13** AI 이미지 생성 기능
- **F-14** AI가 필요한 이미지를 알아서 찾아 추천
- **F-15** 이미지 생성용 프롬프트 자동 작성
- **F-16** 사용자 이미지 직접 업로드
- **F-17** 평소 마음에 드는 이미지를 미리 저장(아카이빙)해 '내 취향 풀' 구성
- **F-18** 타인의 무드보드 이미지를 서로 공유·재사용(사용자 공급 이미지 풀)
- **F-19** 저작권 프리 이미지 API 연동 또는 링크 제공

**보드 제작·편집**

- **F-20** 9칸 격자 등 고정 템플릿 보드
- **F-21** 자유 배치형(비전보드) 캔버스
- **F-22** 보드 type별 다른 배치 방식 제공
- **F-23** 서비스 내 콜라주·이미지 조합 기능
- **F-24** 드래그로 꾸미기(스티커 등) 편집 기능
- **F-25** 완성된 무드보드를 골라 수정·커스텀
- **F-26** 단계별로 실시간 채워지는 무드보드 미리보기
- **F-27** 진행률 표시(상단 % 등)
- **F-28** 완성 화면 사진 조립 애니메이션 연출
- **F-29** 한 줄 코멘트 남기기(내가 만든 흔적)
- **F-30** 가장 나 같은 대표 사진 한 장 픽하기
- **F-31** 워터마크·로고 없는 '느좋' 결과물(서비스 아이덴티티 제거)

**친구·상호작용**

- **F-32** 친구에게 "내 보드 만들어줘" 초대 링크 보내기
- **F-33** 친구가 큐레이션 카드 3~5장 선택으로 보드 만들어주기
- **F-34** "내가 보는 나" vs "친구가 보는 나" 보드 나란히 비교
- **F-35** 같은 주제로 친구들과 취향 비교
- **F-36** 추구미 vs 도달가능미 비교 보드
- **F-37** 친구와 추구미 합쳐보기(합성) 기능
- **F-38** 친구와 보드 공동 편집
- **F-39** 친구 추가 기능
- **F-40** 친구가 보드를 업데이트하면 알림("친구가 업데이트했어요")
- **F-41** 타인 모드 결과에 "선물 도착" 팡 애니메이션 연출
- **F-42** 친구에게 보내는 전달 메시지 추가
- **F-43** 예시 이미지·보드 미리보기로 참여 욕구 유도
- **F-44** 귀엽고 아기자기한 인터랙티브 UI 요소

**공유·확산**

- **F-45** 완성 보드 SNS(인스타 스토리 등) 공유
- **F-46** 카카오톡 공유 썸네일 제공
- **F-47** 공유 링크를 본 사람이 따라 들어와 자기 보드 만들기(유입)

**저장·히스토리·재방문**

- **F-48** 서비스 내 보드 저장(1회성 생성 X)
- **F-49** 보드 히스토리(취향 변화 기록)
- **F-50** 시즌/계절 보드("이번 계절의 나")
- **F-51** 과거 보드 vs 현재 보드로 취향 변화 비교
- **F-52** 계절별·시즌 재방문 알림
- **F-53** 커뮤니티 기능(인기 보드 조회, "봄 보드 조회수 1위 유저 보드 보기" 알림)

## 팀이 추가한 기능

> Agent 목록에 없거나 빠진 기능. 팀이 직접 적는다.

**계정·진입**

- **F-54** 로그인/로그아웃(계정 인증)
- **F-55** 테스트는 로그인 없이 진행, 저장 시점에 로그인 유도(게이팅)
- **F-56** 메인/랜딩 페이지
- **F-57** 내 프로필 이미지 설정

**질문·테스트**

- **F-58** 주제 카테고리 선택(음식·네일·동물 등)
- **F-59** 테스트 결과를 9개 키워드로 도출
- **F-60** 추구미 성향/유형 표기

**보드 편집 모드**

- **F-61** 무드보드 편집 - AI 생성 모드
- **F-62** 무드보드 편집 - 캔버스(자유배치) 모드
- **F-63** 무드보드 편집 - 템플릿 사진 교체 모드
- **F-64** 보드에 글자 쓰기(레터링)
- **F-65** 아날로그 감성 인터랙션(집어넣기·붙이기 등 물리적 느낌)

**저장·관리**

- **F-66** 무드보드 임시저장(초안)
- **F-67** 내 무드보드 목록 보기
- **F-68** 무드보드 삭제
- **F-69** 완성 이미지 파일로 내보내기(다운로드)

**상호작용·탐색**

- **F-70** 무드보드 좋아요
- **F-71** 계정(사용자) 검색
- **F-72** 타인의 무드보드 검색
