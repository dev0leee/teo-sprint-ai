# Step 11. 기능을 페이지로 나누기

> 목적: Step 10에서 모은 **기능을 화면(페이지) 단위로 나눠** 배치한다.

> **Guide:**
>
> 1. Agent가 `docs/10_feature_list.md`의 기능 목록을 읽고 **페이지 구성 초안**을 작성한다.
> 2. 각 페이지에 **어떤 기능(F-xx)** 이 들어가는지 나열한다.
> 3. 팀이 페이지 수·이름·구성을 보며 수정한다.
>
> - Step 10의 기능은 **어느 페이지에든 모두 배치**한다. (이 단계에서 기능을 빼지 않는다)
> - 사용자 흐름이 자연스럽게 이어지도록 페이지를 묶는다. (예: 로그인 → 메인 → 상세)
> - 한 페이지에 기능이 너무 많으면 **페이지를 나눈다.**
> - **우선순위·스펙 아웃(이번에 만들지 않을 기능)** 은 Step 14 `docs/14_priority_matrix.md`에서 정한다.

**프롬프트 예시**

```
docs/10_feature_list.md와 docs/04~09 맥락을 읽고 docs/11_page_routing_split.md를 채워줘.

규칙:
- Step 10의 모든 기능(F-xx)을 어느 페이지에든 배치 (누락 없음)
- 페이지마다 이름·경로·목적·포함 기능 목록을 작성
- 기능 ID(F-xx)와 기능명을 함께 적기
- 해커톤 MVP 기준으로 페이지 수는 과하지 않게 (보통 3~6개)
- "페이지 간 이동 흐름" 섹션도 채우기
- 구조와 상단 Guide는 유지
```

## 페이지 1: 로그인

> **경로:** `/login`  
> **목적:** 계정 인증. 단, 테스트는 로그인 없이 진행하고 **저장 시점에만** 로그인을 유도(게이팅)한다.

- **F-54** 로그인/로그아웃(계정 인증)
- **F-55** 테스트는 로그인 없이 진행, 저장 시점에 로그인 유도(게이팅)
- **F-57** 내 프로필 이미지 설정

## 페이지 2: 메인 (랜딩·홈)

> **경로:** `/`  
> **목적:** 서비스 개념을 알리고 테스트로 진입시키는 허브. 재방문 시 내 보드/히스토리·탐색의 시작점.

- **F-01** 추구미/비전보드/무드보드 개념 온보딩
- **F-03** 완성된 무드보드 예시 골라 시작(엔드 이미지 상상 유도)
- **F-43** 무드보드 생성

## 페이지 3: 테스트 (질문·취향 입력)

> **경로:** `/test`  
> **목적:** 질문/테스트/선택으로 취향을 파악한다. 이미지에서 시작해 텍스트로 구체화하는 연쇄 질문 흐름.

- 질문/테스트/선택
- 추구미 테스트
- 이미지에서 시작해서 텍스트로 구체화
- 큰 갈래 → 구체화

## 페이지 4: 무드보드 생성중

> **경로:** `/generating`  
> **목적:** 테스트 답변 → 프롬프트 변환 → AI 이미지 생성. 진행률·실시간 미리보기로 완성 기대감을 준다.

- 무드보드 생성중
- 실시간으로 채워지는 무드보드 미리보기
- 무드보드 아날로그 인터렉션
- 무드보드 완성 진행 상태 보여주기
- 진행률 표시
- AI 이미지 생성하기
- 이미지 생성 프롬프트

## 페이지 5: 무드보드 (완성·편집·공유)

> **경로:** `/board/:id`  
> **목적:** 완성된 보드를 프리뷰·꾸미고 저장·공유한다.

- 무드보드
- 완성된 무드보드 프리뷰
- 무드보드 임시저장
- 추꾸미기 기능
- 저장
- 완성된 무드보드 꾸미기
  - 펜
  - 스티커
  - 글자쓰기
  - 커스텀
  - 드래그
- 무드보드 공유하기
- 이미지 내보내기
- 무드보드 공유하기 링크 썸네일

## 페이지 6: 히스토리

> **경로:** `/history`  
> **목적:** 내가 만든 무드보드를 모아 보고 관리한다.

- 히스토리
- 내 무드보드 히스토리 보기
- 무드보드 목록 보기
- 무드보드 삭제

## 페이지 간 이동 흐름

> 사용자가 페이지를 어떻게 오가는지 순서대로 적는다.

1. **메인(2)** 진입 → 추구미/무드보드 개념 온보딩·예시 확인 → "테스트 시작"
2. **테스트(3)** 에서 추구미 테스트 진행 → 이미지에서 시작해 텍스트로 구체화(큰 갈래 → 구체화)
3. 답변 완료 → **무드보드 생성중(4)** 으로 이동 → AI 이미지 생성·실시간 미리보기·진행률 확인
4. 생성 완료 → **무드보드(5)** 에서 프리뷰 확인 → 꾸미기(펜·스티커·글자·드래그·커스텀)
5. 저장/공유 시도 → 미로그인 시 **로그인(1)** 으로 게이팅 → 인증 후 5로 복귀(임시저장은 로그인 전에도 가능)
6. 저장/내보내기·링크 공유 후 → **히스토리(6)** 에서 내 무드보드 목록 재열람·삭제 관리
7. **히스토리(6)** 또는 **메인(2)** 에서 다시 **테스트(3)** 로 진입 → 새 무드보드 생성 루프
