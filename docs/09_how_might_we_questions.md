# Step 09. "어떻게 하면 ~" 질문 만들기

> 목적: 서비스 방향을 맞춘 뒤, **더 구체적인 생각**으로 수렴한다.

> **Guide:**  
> 1. Agent가 `docs/04`, `docs/06`, `docs/08`을 읽고 **"어떻게 하면 ~ 할 수 있을까?"** 형태의 질문을 만든다. 직접 질문을 추가해도 좋다.
> 2. 팀이 질문을 보며 답을 이야기한다.
> 3. 각 질문 아래 **답**을 적는다. (타이머 ON)  
> - 질문은 **많을수록** 좋고, 하고 싶은 말이 많은 질문이 좋다.  
> - **실제로 어떻게 할지** 떠올릴 수 있는 질문을 한다. (예: 사용자를 어떻게 모을까? → 너무 넓음)  
> - 답은 **질문과 같은 말 반복·뭉뚱그린 표현 금지**. 구체적인 문장으로 쓴다.  
>   - (X) 어떻게 하면 사용자가 이 기능을 더 잘 쓸까? → 사용자들이 더 쉽게 쓰게 만들면 될 것 같아요  
> - 참고한 서비스·자료가 있으면 답에 적는다.

**프롬프트 예시**

```
docs/04_service_goal_target_value.md, docs/06_pain_wow_points.md, docs/08_word_cloud.md를 읽고
docs/09_how_might_we_questions.md의 "Agent가 만든 질문" 섹션만 채워줘.

질문 규칙:
- "어떻게 하면 ~ 할 수 있을까?" 형태
- docs/04 목적·핵심 가치, docs/06 불편한 점·좋았던 점, docs/08 키워드를 골고루 반영
- 실제 방법·아이디어가 떠오를 수 있는 질문 (너무 넓은 "사람 모으기"류 X)
- 최소 10개 이상, 서로 겹치지 않게
- 각 질문 아래 답 칸은 비워둬 (팀이 채움)
- "팀이 추가한 질문" 섹션은 건드리지 마
- 구조와 상단 Guide는 유지
```

## 팀이 채운 질문

> 테마별로 묶어 정리했다. 각 질문 아래는 팀 논의 답변(몌굥·녕녕 의견 포함).

### 테마 A. 키워드·질문

**질문 1: 어떻게 하면 "무슨 키워드로 검색하지?"라는 막막함 없이, 사용자가 첫 무드보드를 완성까지 가게 할 수 있을까?**

- 몌굥: 검색 vs 심리테스트
- 녕녕: 테스트 or 검색
- 키워드 검색 → 검색 결과가 무엇일지: 사진, 가치관
- 녕녕: 글자로 할 수도 있고, UI/그림으로 보여줄 수도 있음

**질문 2: 어떻게 하면 가벼운 질문 몇 개의 결과를 사용자가 "이게 나 같다"고 납득하게 만들 수 있을까?**

- 가벼운 질문 = 키워드와 역할이 같음
- 녕녕: 여러 개의 큰 갈래에서 점점 구체화·좁혀지도록
- 결과가 중요!!!
- 매니아적인 요소가 있으므로 시간이 길어도 괜찮지 않을까?!

**질문 3: 어떻게 하면 "나" 모드 질문이 더 내면적이고 솔직한 톤으로 느껴지게 할 수 있을까?**

- 과정이 좀 길더라도(시간 소요가 있어도) 결과가 만족스러우면 사용
- 녕녕: 심도 있는 질문, 나를 잘 꺼낼 수 있는 질문을 줘야 할 듯 (AI에게 심리상담사 역할 부여하고 뽑아달라고 하기)

**질문 4: 어떻게 하면 "타인" 모드 질문이 추측·관찰의 재미를 살리면서도 무례하지 않게 만들 수 있을까?**

- 트레이드오프: 심리테스트 → 가치·자기 발견 → 여정이 길어짐 / 검색 → 내면적으로 솔직하지 못함
- 몌굥: AI가 재미 차원에서 괜찮을 수 있고 무거운 flow를 생성 단계에서 잡아줄 수 있을 듯 (이걸 잘 하려면 AI가 잘 잡아줘야 함)
- 녕녕: 우리끼리 서로 받아보고 싶은 질문을 이야기해보기?!

### 테마 B. 이미지 확보

**질문 5: 어떻게 하면 저작권 걱정 없이 감성적인 사진 풀을 빠르게 확보할 수 있을까?**

- API 확인 필요!!! 아니면 링크만 제공?!
- AI가 생성 → 프롬프트를 잘 적어야 한다?
- 저작권 걱정이 없으려면 사용자에게 이미지를 직접 받거나 AI가 사용되어야 할 듯
- 평소에 이미지 아카이빙
- 서비스 내에서 타인의 무드보드를 보고 그 이미지를 사용할 수 있다면? → 검증된 퀄리티 이미지 사용 → 저작권 책임 위임
- 사용자가 평소 이미지를 모아두게 하기

**질문 6: 어떻게 하면 AI가 필요한 이미지를 적절하게 찾거나 생성해줄 수 있을까?**

- AI 도입, 프롬프트 기깔나게 작성하기
- 무료 이미지 사용 + 보완할 부분만 AI가?
- 타인이 쓴 이미지를 서로 공유. 이미지는 사용자 공급 기반 → 처음엔 어려워도 장기적으로 가능

**질문 7: 어떻게 하면 평소 마주친 마음에 드는 이미지를 부담 없이 미리 모아두게 하고, 보드 만들 때 그 '내 취향 풀'에서 바로 꺼내 쓰게 할 수 있을까?**

- 미리 모아둘지 말지는 사용자 영역?! → 바로 꺼내 쓰기 쉽게 하는 게 중요할 듯

### 테마 C. 이탈률

**질문 8: 어떻게 하면 진입한 사람이 복잡한 플로우 없이 바로 핵심 기능(내 보드 만들기)까지 도달해 이탈하지 않게 할 수 있을까?**

**질문 9: 어떻게 하면 친구와 같이 보드 만드는 재미를 느끼게 할까?**

- 친구 것 만들어주기
- 친구랑 비교 (서비스 내는 아니고 셀프)
- 친구랑 공동 편집
- "선물이 도착했습니다~!" 팡~ 애니메이션?!
- 귀엽고 아기자기한 요소

**질문 10: 어떻게 하면 사용자가 한 번 만들고 끝내지 않고, 시즌·계기마다 다시 돌아와 취향 변화를 쌓아가게 할 수 있을까?**

- 진행률 체감시켜주기 (상단에 50% 진행)
- 무드보드가 실시간으로 채워지는 느낌을 줘서 다음으로 넘어가게?
- 처음에 이미 완성된 무드보드들 중에서 선택해 들어간다면? → 엔드 이미지를 상상하기 쉽고 기대돼서 완성 전 이탈이 줄 듯
- 계절별 알림 기능 추가
- 커뮤니티 기능 ("봄 보드 조회수가 가장 높은 유저의 보드를 확인해보세요!" 알림)

**질문 11: 어떻게 하면 타인 모드에서 나온 결과가 "선물처럼" 느껴지게 설계할 수 있을까?**

- 인터랙티브, UI에 힘을 주자!
- 친구 추가 기능이 있고, 친구가 무드보드를 업데이트하면 따라 들어와 자기도 수정하지 않을까
- 완성된 무드보드 중에서 선택!!해서 수정할 수 있도록
- 단계별로 제공!
- "친구가 업데이트했어요!" 알림
- 뿌듯함

### 테마 D. 공유

**질문 12: 어떻게 하면 쉽게 만들어주면서도 "내가 만든 보드"라는 뿌듯함·애착이 사라지지 않게 할 수 있을까?**

- 결과에 "내가 만든 흔적" 남기기 → 한 줄 코멘트 추가, 가장 나 같은 대표 사진 한 장 픽하기
- 완성 화면을 임팩트 있게 → 사진들이 하나씩 자리를 찾아 조립되는 짧은 연출 → 실제 노동을 대체하는 심리적 장치가 될 수 있지 않을까
- 내 이야기가 담긴다면 애착이 더해질 듯 → 질문 자체가 내 얘기처럼 느껴지게
- 직접 편집이 아니라면, 이미지 자원이 내가 업로드한 사진이라면?
- 내가 찍은 사진이 일부라도 들어가면 애착이 생길 듯
- 너무 쉽게만 만들지는 않도록

**질문 13: 어떻게 하면 친구가 부담 없이 "너를 위한 보드 만들기"에 참여하게 할 수 있을까?**

- 이쁘기만 한 게 참여 욕구를 일으킬까? 위트가 필요하지 않을지?
- 예시 이미지 제공해서 "와, 해보고 싶다!"
- 친구의 전달 메시지 추가
- 내가 알콩달콩 꾸미는 느낌
- 카카오톡 공유 썸네일
- 질문에 힘을 주자! 결과가 정확하게 나올 수 있도록!

**질문 14: 어떻게 하면 공유 링크를 본 사람이 구경에서 그치지 않고 직접 자기 보드를 만드는 데까지 가게 할 수 있을까?**

- 유입량 증가하려면 좋은 마케팅하기
- 아날로그틱한, 내가 만든 것 같은, 간접 체험
- 자동화 느낌 추가 가능

**질문 15: 어떻게 하면 사용자가 완성한 보드를 스스로 SNS에 자랑·공유하고 싶게 만들 수 있을까?**

- 예쁘게 만들기
- 차별화

### 테마 E. 차별화

**질문 16: 어떻게 하면 우리 '추구미 보드'가 그냥 이미지 아카이빙이 아니라, 다른 '무엇'을 담는 서비스로 차별화될 수 있을까?**

- 이미지 아카이빙보다는 자기 발견, 느낌 좋은, 쓰고 싶은 UI에 집중
- 인생네컷처럼 고유한 템플릿으로 반대로 한정시켜버리기
- 템플릿은 고정하되 다른 무드를 보여주면 비교하기도 수월함
- 고정된 영역이 이 서비스의 아이덴티티가 될 수 있을 듯!
- 너무 커스텀하기보다 고정된 것을 바탕으로 디자인 디테일을 더 가져갈 수 있는 부분이 있음
- 아카이빙이 아니라, 나를 알아가는 결과물
- 이미지 보관에 집중하되 만드는 과정 자체에 재미를 주는
