# Step 13. 스케치 및 결정

> 목적: 화면 설계안을 **각자 그려본 뒤** 팀이 맞추고, **최종 스케치만** 남긴다.

> **Guide:** (Step 02처럼 **대면 워크숍**으로 진행. Agent가 대신 채우지 않는다.)  
> 1. **각자** AI(v0, Bolt.new, Cursor 등)에게 `docs/11`·`docs/12`을 참고해 **페이지별 화면 스케치**를 그려본다.  
> 2. 팀이 스케치를 **공유**하고, 레이아웃·흐름·느낌을 **이야기**한다.  
> 3. 합의된 **최종 스케치**만 이 문서에 등록한다.  
> - 스케치 생성·비교는 각자·팀에서 한다. 이 문서에는 **결정 결과**만 적는다.  
> - v0·Bolt.new 등에 넘길 문장은 `docs/11`·`docs/12`을 참고해 각자 AI에게 요청하면 된다.  
> - 여러 후보 중 **팀이 고른 것**만 적는다. 개인 스케치 전부를 문서에 옮기지 않는다.

**초안 대비 달라진 점**

- **편집(5)과 완성·공유(6)를 별도 화면으로 분리** — 편집 화면에 내보내기·공유가 함께 있고, 완성 화면은 결과(보드 + 무드 성향) 감상에 집중
- **히스토리 페이지는 이번 스케치에서 제외** (MVP 범위 조정)
- 로그인에 **"나중에 할게요"** 건너뛰기 동선 명시 — 저장 게이팅 원칙 유지
- 테스트 선택지는 **이미지 / 텍스트 / 이미지+텍스트** 혼합 타입 + 옵션 텍스트 입력

## 합의 사항

> 스케치 토론에서 팀이 맞춘 내용

- **디자인 방향:** 감성·아날로그 무드의 따뜻한 인터랙티브 UI. 이미지가 주인공이고 텍스트·크롬은 최소화하며, 완성까지의 기대감(진행률·실시간 미리보기)을 연출한다.
- **공통 UI 규칙:** (색, 버튼, 레이아웃 등) 모바일 세로 단일 컬럼 우선 / 주요 행동은 하단 고정 CTA 1개 / 이미지는 라운드 8px 카드·썸네일 그리드로 통일 / 인터랙션 요소는 accent 색으로만 구분.
- **구현 시 유의:** 테스트는 비로그인 진행 후 저장 시점 게이팅 / 생성 지연 대비 임시저장·진행 상태 필수 / 공유 링크 썸네일(OG) 미리 설계 / 히스토리 삭제는 되돌리기 확인 절차.
