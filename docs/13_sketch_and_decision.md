# Step 13. 스케치 및 결정

> 목적: 화면 설계안을 **각자 그려본 뒤** 팀이 맞추고, **최종 스케치만** 남긴다.

> **Guide:** (Step 02처럼 **대면 워크숍**으로 진행. Agent가 대신 채우지 않는다.)  
> 1. **각자** AI(v0, Bolt.new, Cursor 등)에게 `docs/11`·`docs/12`을 참고해 **페이지별 화면 스케치**를 그려본다.  
> 2. 팀이 스케치를 **공유**하고, 레이아웃·흐름·느낌을 **이야기**한다.  
> 3. 합의된 **최종 스케치**만 이 문서에 등록한다.  
> - 스케치 생성·비교는 각자·팀에서 한다. 이 문서에는 **결정 결과**만 적는다.  
> - v0·Bolt.new 등에 넘길 문장은 `docs/11`·`docs/12`을 참고해 각자 AI에게 요청하면 된다.  
> - 여러 후보 중 **팀이 고른 것**만 적는다. 개인 스케치 전부를 문서에 옮기지 않는다.

## 최종 스케치

> 페이지별로 팀이 선택한 스케치. 링크·스크린샷 경로·한 줄 설명 중 편한 방식으로.
>
> **전체 와이어프레임:** [`docs/assets/13_sketch_wireframe.html`](assets/13_sketch_wireframe.html) — 6개 페이지 저해상도 와이어프레임 (Step 11·12 기반)

### 페이지 1 — 로그인 (`/login`)

> **선택:** 저장 게이팅 안 (테스트는 비로그인 진행, 저장 시점에만 로그인)  
> **링크·이미지:** `docs/assets/13_sketch_wireframe.html` (screen 1)  
> **한 줄 요약:** 프로필 + 소셜 로그인(카카오/구글), 저장 유도 문구 중심의 최소 화면

### 페이지 2 — 메인 (`/`)

> **선택:** 온보딩 + 예시 갤러리 + 하단 CTA 안  
> **링크·이미지:** `docs/assets/13_sketch_wireframe.html` (screen 2)  
> **한 줄 요약:** 개념 소개와 완성 예시로 기대를 만들고 "무드보드 만들기"로 진입

### 페이지 3 — 테스트 (`/test`)

> **선택:** 이미지 선택 → 텍스트 구체화, 진행률 표시 안  
> **링크·이미지:** `docs/assets/13_sketch_wireframe.html` (screen 3)  
> **한 줄 요약:** 이미지 그리드에서 골라 큰 갈래 → 구체화로 취향을 좁히는 연쇄 질문

### 페이지 4 — 무드보드 생성중 (`/generating`)

> **선택:** 진행률 + 실시간 채워지는 미리보기 안  
> **링크·이미지:** `docs/assets/13_sketch_wireframe.html` (screen 4)  
> **한 줄 요약:** % 진행률과 칸이 하나씩 채워지는 미리보기로 완성 기대감 연출

### 페이지 5 — 무드보드 (`/board/:id`)

> **선택:** 프리뷰 + 꾸미기 툴바 + 저장/공유 안  
> **링크·이미지:** `docs/assets/13_sketch_wireframe.html` (screen 5)  
> **한 줄 요약:** 완성 프리뷰 위에 펜·스티커·글자·드래그 툴바, 하단에 저장·내보내기·공유

### 페이지 6 — 히스토리 (`/history`)

> **선택:** 썸네일 그리드 + 항목별 삭제 안  
> **링크·이미지:** `docs/assets/13_sketch_wireframe.html` (screen 6)  
> **한 줄 요약:** 내 무드보드 목록을 썸네일로 보고 삭제·재열람, 새로 만들기로 루프

## 합의 사항

> 스케치 토론에서 팀이 맞춘 내용

- **디자인 방향:** 감성·아날로그 무드의 따뜻한 인터랙티브 UI. 이미지가 주인공이고 텍스트·크롬은 최소화하며, 완성까지의 기대감(진행률·실시간 미리보기)을 연출한다.
- **공통 UI 규칙:** (색, 버튼, 레이아웃 등) 모바일 세로 단일 컬럼 우선 / 주요 행동은 하단 고정 CTA 1개 / 이미지는 라운드 8px 카드·썸네일 그리드로 통일 / 인터랙션 요소는 accent 색으로만 구분.
- **구현 시 유의:** 테스트는 비로그인 진행 후 저장 시점 게이팅 / 생성 지연 대비 임시저장·진행 상태 필수 / 공유 링크 썸네일(OG) 미리 설계 / 히스토리 삭제는 되돌리기 확인 절차.
