# Step 12. 고객 여정 맵

> 목적: 팀이 **우리 서비스**에서 사용자가 **어떤 순서로 무엇을 하는지** 같은 그림으로 이해한다. (Step 05는 기존·대체 방식, Step 12는 우리 서비스)

> **Guide:**  
> 1. Agent가 `docs/11_page_routing_split.md`만 읽고 **고객 여정 맵 초안**을 작성한다.  
> 2. 팀이 표를 보며 단계·행동·감정이 맞는지 이야기하고 수정한다.  
> - **데모에서 보여줄 핵심 여정 1개**를 중심으로 쓴다.  
> - Step 11의 **페이지·기능·이동 흐름**을 바탕으로 단계를 구성한다.  
> - 각 단계는 **구체적인 행동**으로 쓴다. (예: "앱을 연다" → "일정을 입력하고 추천 버튼을 누른다")  
> - Step 05처럼 **표 한 벌**로 정리한다. 팀원 모두가 흐름을 따라갈 수 있어야 한다.

**프롬프트 예시**

```
docs/11_page_routing_split.md만 읽고 docs/12_customer_journey_map.md를 채워줘.

규칙:
- Step 11의 페이지 구성·기능 목록·페이지 간 이동 흐름만 참고
- 우리 서비스 기준의 핵심 여정 1개 (데모 가능한 흐름)
- 단계 | 사용자 행동 | 생각/감정 | 해당 페이지 | 사용 기능(F-xx) 표 작성
- Step 11에 있는 페이지와 F-xx를 빠짐없이 연결
- 구조와 상단 Guide는 유지
```

## 여정 요약

> **누가** → **무엇을 하려고** → **어떤 순서로** → **무엇을 얻는다**

- **나의 취향(추구미)을 시각화하고 싶은 사용자**가 → **내 무드보드를 만들려고** → **메인에서 개념을 보고 → 테스트로 취향을 파악 → AI가 무드보드를 생성 → 꾸미고 저장·공유 → 히스토리에서 재열람**하는 순서로 → **나를 대표하는 무드보드와 공유 링크를 얻는다.**

## 고객 여정

> MVP·데모에서 보여줄 주 사용자 흐름

| 단계 | 사용자 행동 | 생각/감정 | 해당 페이지 | 사용 기능 |
|---|---|---|---|---|
| 1. 진입·온보딩 | 메인에 들어와 추구미/무드보드 개념과 완성 예시를 보고 "테스트 시작"을 누른다 | "이게 뭐지? 예시 보니 나도 만들어보고 싶다" | 페이지 2: 메인 (`/`) | F-01 개념 온보딩, F-03 예시 골라 시작, F-43 무드보드 생성 |
| 2. 취향 테스트 | 추구미 테스트를 진행하며 이미지에서 시작해 텍스트로 취향을 구체화한다 (큰 갈래 → 구체화) | "고르는 재미가 있네, 내 취향이 점점 잡힌다" | 페이지 3: 테스트 (`/test`) | 추구미 테스트, 질문/선택, 이미지→텍스트 구체화 |
| 3. 생성 대기 | 답변이 프롬프트로 변환되고 AI가 이미지를 생성하는 동안 실시간 미리보기·진행률을 본다 | "실시간으로 채워지니 기대된다" | 페이지 4: 무드보드 생성중 (`/generating`) | AI 이미지 생성, 이미지 생성 프롬프트, 실시간 미리보기, 진행률 표시, 아날로그 인터렉션 |
| 4. 확인·꾸미기 | 완성된 무드보드 프리뷰를 보고 펜·스티커·글자·드래그로 커스텀한다 | "오 내 취향 그대로다, 더 꾸며보자" | 페이지 5: 무드보드 (`/board/:id`) | 완성 프리뷰, 꾸미기(펜·스티커·글자쓰기·커스텀·드래그) |
| 5. 저장·공유 | 저장/내보내기를 하고 공유 링크(썸네일)를 만든다. 미로그인 시 로그인으로 게이팅된다 | "간직하고 친구한테도 보여주고 싶다" | 페이지 5 → 페이지 1: 로그인 (`/login`) | 임시저장, 저장, 이미지 내보내기, 공유 링크 썸네일 / F-54 로그인, F-55 저장 게이팅 |
| 6. 재열람·관리 | 히스토리에서 내 무드보드 목록을 다시 보고 관리(삭제)한다. 다시 테스트로 새 보드를 만든다 | "내 기록이 쌓인다, 또 만들어볼까" | 페이지 6: 히스토리 (`/history`) → 페이지 3 | 히스토리 목록 보기, 무드보드 삭제 |
