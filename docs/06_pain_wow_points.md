# Step 06. PainPoint / WowPoint

> 목적: `docs/05` 기존 여정에서 Pain·Wow를 추출하고, 우리 서비스의 개선·차별 방향으로 정리한다.

> **Guide:** 기존 여정에서 사용자가 **불편했던(Pain)** 점은 우리 서비스에서 개선하고, **좋았던(Wow)** 점은 가져와 매력 있고 가치 있는 서비스를 만든다. `docs/05`의 모든 여정·단계를 검토해 Pain/Wow를 빠짐없이 나열한다.

**프롬프트 예시**

```
docs/05_existing_customer_journey.md를 읽고 docs/06_pain_wow_points.md를 채워줘.
각 여정·단계에서 사용자가 불편했던 Pain, 좋았던 Wow를 모두 추출해 나열해줘.
하나로 합치거나 우선순위를 정하지 말고, 발견한 항목을 빠짐없이 bullet로 정리해줘.
구조와 상단 Guide는 유지하고, 본문만 채워줘.
```

## 기존 사람들은 목적을 달성 하기 위해서 어떠한 서비스와 단계들을 이용해서 해결하고 있나요?

### 클로드

- **핀터레스트로 추구미/비전보드 직접 만들기** (여정 1) — 영감(SNS·릴스) → 키워드 검색 → 핀 저장·수집 → 별도 앱(캔바·포토샵·모바일 편집기)으로 콜라주 배치 → 캡처 저장 → 인스타 스토리·프로필 공유.
- **MBTI·심리/성격 테스트로 나 알아보기** (여정 2) — 친구가 공유한 테스트 링크 접함 → 질문 응답 → 유형·키워드 결과 확인 → 결과 이미지 캡처·공유 → 친구와 결과 비교 → 며칠 뒤 잊고 이탈.
- **롤링페이퍼·익명질문으로 "친구가 보는 나" 모으기** (여정 3) — 생일·MT·송별 계기 → 종이 롤링페이퍼·구글폼·인스타 익명질문 스티커 준비 → 친구들에게 부탁 → 텍스트 위주 답변 수집 → 수작업 취합·정리 → 캡처·종이 보관.
- **친구가 대신 만들어줌** (녕녕 여정 1) — 부탁 받음 → 추구미 보드가 뭔지 검색·파악 → 키워드 설정 → 핀터레스트·구글·네이버에서 키워드당 이미지 검색 → 후보 선별 → 조합 앱 탐색·설치 → 이미지 조합 → 친구에게 공유.
- **다른 취향찾기/심리 플랫폼 이용** (몌굥 여정 1·2) — 핀터레스트 대체제로 취향찾기 플랫폼 사용, 친구 추구미도 플랫폼으로 확인.
- **수제 제작(오려 붙이기)** (몌굥 여정 1) — 이미지를 직접 오리고 붙여 아날로그로 무드보드 제작.
- **추구미 변화 수동 기록** (몌굥 여정 3) — 변해가는 취향을 수동으로 기록하며 흐름을 남기려 시도.
- 참고: 실제로 핀터레스트에서 **outfit / food / nails / quote** 같은 키워드를 검색해 이미지를 추구미로 삼는 패턴이 관찰됨(몌굥).

### 류하

- 내가 원하는 이미지를 찾고 검색해서 이미지를 저장한다.

- 콜라주 이용해서 본인이 원하는대로 배치한다.

- 친구에게 직접 sns를 통해 전달/ 본인 스토리에공유함.

### 몌굥

- 추구미 보드를 만들고 싶음
  - 1.추구미 보드를 만들고 싶음 → 핀터레스트 이용 → 핀터레스트 검색
    - a.어떤 키워드로 검색해야 원하는 이미지를 찾을 수 있는지 모름
    - b.보드에 이미지를 쌓고나서 한눈에 확인하기 어려움
  - 2.인스타에서 이미지를 찾아봄
    - a.평소에는 예쁜게 잘 보이는데, 막상 필요할 때 못찾겠음
  - 3.수제로 만듬 → 이미지 오리고 붙이고 만들기
    - a.피곤(체력이슈)
    - b.보관이 어려움
    - c.하지만 만드는 과정에서 원하는 추구미 윤곽이 정리되는 느낌이 있음
  - 4.다른 취향찾기 플랫폼을 이용
    - a.딱 내가 원하는 느낌이 아님
- 친구의 추구미가 알고 싶음
  - 친구에게 직접 물어보기 → 답변 회피 당함
  - 취향 찾기 플랫폼 이용 → 단편적인 결과물을 받음
- 변해가는 추구미를 알고 싶음
  - 1.수동으로 기록
    - a.꾸준히 기록하지 않아서 모름
    - b.기록 유실됨
- 핀터레스트 보드 만들기 → 보드 기준으로 추천 이미지에서 핀을 늘림
- 핀터레스트에 콜라쥬 기능이 있음
- 인스타 추구미 or 취향만 올리는 계정 만들기

### 녕녕

- 추구미/비전보드 제작중
  - 영감 찾기 : 텍스트로 내가 어떤 사람인지 써보기, SNS에서 다른 사람들의 예시 찾아보기, SNS 사진들로부터 영감 얻기
  - 이미지 찾기 : SNS 활용
  - 사진 콜라주 배치 : 별도 서비스 활용
  - 지인 공유 → 공유 기능 사용. SNS로 직접 전달
  - 날짜별 보드 → 내 사진첩 속 저장..

## PainPoint

### 클로드

**핀터레스트로 직접 만들기 (여정 1)**

- 검색: 내 느낌을 어떤 키워드로 검색해야 할지 애매해 막막함.
- 이미지 수집: 이미지는 많은데 내 취향이 맞는지 확신이 안 서는 선택 피로.
- 배치·정리: 예쁘게 배치하는 게 제일 어렵고 시간이 오래 걸려 지침(별도 앱까지 오가야 함).
- 완성: 몇 시간을 썼는데 이게 정말 '나'를 표현한 건지 애매한 성취감.
- 공유: 반응은 좋아도 다시 만들 엄두가 안 나는 높은 진입장벽.

**MBTI·심리 테스트 (여정 2)**

- 결과 확인: 결과가 다 비슷비슷하게 느껴지는 아쉬움.
- 친구와 비교: 잠깐 얘기하고 끝나는 일회성·휘발성.
- 이탈: 재밌었지만 '나'로 남는 결과물이 없어 며칠 뒤면 잊힘.

**롤링페이퍼·익명질문 (여정 3)**

- 도구 준비: 매번 도구(종이·폼·스티커)가 제각각이고 세팅이 번거로움.
- 요청: 부탁하기 민망하고 참여율이 걱정되는 부담.
- 수집: 텍스트만 오고 '무드'나 이미지로는 모이지 않는 한계.
- 취합: 흩어진 답을 전부 수작업으로 정리해야 하는 수고.
- 간직·공유: 다시 보기 어렵고 내 보드와 비교할 방법이 없는 휘발·단절.

**친구가 대신 만들어줌 (녕녕 여정 1)**

- 생소함: 추구미 보드가 뭔지부터 검색해 파악해야 함.
- 이미지 찾기: 검색처(핀터레스트·구글·네이버)가 제각각이고 고를 게 너무 많은 수고.
- 조합 도구: 사진 조합 앱을 뭘 써야 할지 몰라 탐색·설치하는 인지부채.
- 소진: 완성까지 시간이 너무 오래 걸려 "다신 못 하겠다"는 완전 소진(지속 불가능).

**내 추구미 보드 만들기 (몌굥 여정 1)**

- 검색: 원하는 이미지가 나오는 키워드를 모르겠는 막막함.
- 수집: 쌓고 나면 한눈에 확인·정리가 안 됨.
- 탐색의 아이러니: 평소엔 예쁜 게 잘 보이는데 막상 필요할 때 못 찾음.
- 수제 제작: 체력 소모가 크고 보관이 어려움.
- 대체 플랫폼: 딱 내가 원하는 느낌이 아니라 아쉬움.

**친구의 추구미가 알고 싶음 (몌굥 여정 2)**

- 직접 질문: 답변을 회피당해 무안·막힘.
- 플랫폼: 단편적인 결과물만 받아 부족함.

**변해가는 추구미를 알고 싶음 (몌굥 여정 3)**

- 지속 실패: 꾸준히 기록하지 못해 결국 흐름을 모름.
- 기록 유실: 남긴 기록마저 유실돼 남는 게 없는 허무함.

**공통 관통 Pain**

- 이미지 찾기 자체가 힘들고, 보관도 힘들고, 꾸준히 하기도 힘들다.
- 어렵게 만든 결과물이 정작 '이쁘지' 않을 수 있다(추구미 트렌드 미달).
- 어느 방식이든 남는 결과물이 휘발되고, "내가 보는 나"와 "친구가 보는 나"를 한 곳에서 비교할 수단이 없다.

### 류하

- 시간이 오래 걸린다.

- 내가 원하는 이미지를 찾기가 어렵다.

### 몌굥

- 아쉬운 점 : 이미지 찾는게 힘듬. 보관도 힘듬. 꾸준히도 힘듬. 그게 이쁘지가 않음
- 기회 : 이미지를 찾기 쉽게, 혹은 필요한 이미지를 알아서 찾기 or 생성, 무드보드 1회성 생성X, 서비스내 저장, 보드에서 서비스 아이덴티티를 지워내, 온전한 느좋 느낌 주기

### 녕녕

- 생소함(추구생보드, 추구미보드, 비전보드의 개념을 모르는 사람일 수도 있음)
  개념을 직관적으로 알려주기
- 키워드 설정 막막함
  - 내가 어떤 걸 추구하는지 잘 모를 수 있음. 비전 여정 or 추구생 여정 or 추구미 여정(test)을 통해 질문으로 같이 알아가보는 과정을 추가
    - test 레퍼런스 : 🌿 나의 가치 여정 내 인생을 관통하는 5가지 단어(word-journey.site)
- 이미지 찾기 힘듦
  - 오히려 이 부분은 계속 Pain point 로 가져가는 것이 ‘나만의 보드'를 만들었다는 뿌듯함을 가져다 줄 수 있을 듯. 쉽지만(다른 기능들이 제공되니) 쉽지 않은(이쁘게, 차별화되게 만들려고) 느낌
  - 만약 찾는 게 힘들 것 같다면, AI 이미지 생성 기능을 제공..?
  - 평소엔 잘 보이는 예쁜 사진들이 막상 필요할 땐 안 보이니, 사진을 저장해둘 수 있는 기능..?
- 이미지 배치시 별도 서비스 검색, 설치, 사용 해야함
  - 서비스 내에서 제공함으로써 손 쉽게 배치(격자 모양은 쉽게 배치할 수 있고, 비전보드는 자유롭게 배치할 수 있도록 함. type 마다 다른 배치방법 제공..?)
- 보드 히스토리
  - 분산되어 있어 기록 유실 가능성 있음. 히스토리 기능 제공
- 지인과 별도 SNS로 공유해야함
  - 서비스 내 지인과 같이 해볼 수 있는 환경 제공

## WowPoint

### 클로드

**핀터레스트로 직접 만들기 (여정 1)**

- 계기·영감: 남의 보드를 보면 "나도 만들고 싶다"는 기대·설렘이 즉시 생김(강한 동기).
- 공유: 완성해서 올리면 반응이 좋음(자랑·표현 욕구 충족).

**MBTI·심리 테스트 (여정 2)**

- 발견: 친구 링크만으로 가볍게 시작되는 낮은 진입장벽·호기심.
- 응답: 질문이 쉽고 빠르게 끝나 몰입됨.
- 결과 확인: "오 은근 맞네" 하는 공감의 순간.
- 저장·공유: 친구들과 결과를 비교하고 싶어지는 자연스러운 공유 욕구.

**롤링페이퍼·익명질문 (여정 3)**

- 계기: "친구들은 나를 어떻게 볼까?"라는 궁금·설렘이 강한 참여 동기.
- 간직: 친구들의 진심이 담겨 따뜻하고 소중한 감정이 남음.

**친구가 대신 만들어줌 (녕녕 여정 1)**

- 키워드 정리: 사진이 막막할 때 키워드부터 잡으면 정리가 시작됨(질문·키워드 기반 접근의 효용).
- 공유: 완성본을 받은 친구가 웃고 좋아해 뿌듯·보람이 큼.
- 확장 힌트: "AI로 이미지를 생성하는 방법"이면 이 소진을 줄일 수 있다는 가능성.

**내 추구미 보드 만들기 (몌굥 여정 1)**

- 수제 제작: 손으로 만드는 과정에서 원하는 추구미의 윤곽이 정리되는 느낌(과정의 발견 가치).
- 어렵게 만든 결과물에서 오는 뿌듯함·소중함·한땀한땀의 애착.

**변해가는 추구미를 알고 싶음 (몌굥 여정 3)**

- 기대: 기록해두면 취향의 흐름·변화를 볼 수 있으리라는 기대(시즌·주기 컨셉의 씨앗).

**공통 관통 Wow**

- "나를 이미지로 표현하고 싶다", "친구가 보는 나가 궁금하다"는 욕구 자체가 강력한 참여 동력.
- 질문·키워드로 접근하면 막막함이 정리되고, 결과물을 만들면 공유·자랑으로 이어짐.
- 함께할 때(친구가 만들어줌, 결과 비교) 재미와 보람이 커짐 → "혼자도·함께도 재미"의 근거.

### 류하

- 몇 번의 선택으로 사진 한장에 담긴 나 !!

- 질문 몇개에 대답한건데 → 노력대비 결과물의 격차가 감동포인트

- 친구끼리 공유하기

- 이미지 퀄리티와 무드보드의 조화가 포인트일듯.. → 어떻게 가져오는게 좋을까

### 몌굥

- 공유를 타고 들어왔을 때, 나와 친구 간의 상호작용
  - 내 친구가 짜주는 추구미
  - 추구미 vs 도달가능미
  - 같은 주제로 친구들과 다른 취향 비교
- 기회
  - 이미지를 찾기 쉽게
  - 혹은 필요한 이미지를 알아서 찾기 or 생성
  - 무드보드 1회성 생성 X, 서비스 내 저장
  - 보드에서 서비스 아이덴티티를 지워내, 온전한 '느좋' 느낌 주기
  - 공유의 재미
- 세대 인사이트
  - 추구미는 Z세대와 알파세대를 통칭하는 '잘파세대'가 압도적으로 주도하고 있음
  - Z세대는 내 취향의 정교화, 알파세대는 추구미의 시각적 생산에 몰두
- '추구미' 보드의 차별점?
  - '추구미'에 대한 차별점이 없으면 이미지 아카이빙 서비스에 이름만 추구미 보드가 됨 → 무엇을 아카이빙 하는 서비스인가, '무엇'이 날카롭고 첨예해야 한다.
  - 짚어야 할 질문: 추구미에 대한 정의 / 추구미 보드가 기존 이미지 아카이빙과의 차이점 / 비전보드·무드보드와의 차이?
- 이미지를 플랫폼에서 선택하지 않고 프롬프트만 만든다면? (보드·이미지를 AI로 생성)
  - 서비스에서 프롬프트만 만들고 실제 이미지 생성은 아웃소싱 or 서비스 내 생성까지
  - 장점: 이미지 편집·가공 피로 감소 / 직접 편집보다 시간이 적게 듦
  - 유의: 사용자가 적확하게 원하는 보드가 아닐 수 있음 → 단, 심리테스트처럼 가벼운 서비스를 지향한다면 단점은 아님
  - 조건: AI 이미지 생성 프롬프트의 퀄리티가 보장되어야 함
  - 아웃소싱의 경우 사용자가 다시 서비스에 콘텐츠를 업로드해야 함 → 가능하면 서비스 내에서 생성 가능한 게 좋음
